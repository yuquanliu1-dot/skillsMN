# 🤖 AI 功能使用指南

## 📋 概述

skillsMN 集成了 AI 辅助功能，可以帮助你快速生成、修改和改进技能内容。

## 🚀 快速开始

### 1️⃣ 配置 AI（首次使用）

#### 步骤 1: 打开设置
- 点击右上角 **⚙️ 设置图标**
- 进入 **AI Configuration** 标签

#### 步骤 2: 配置 AI

**使用智谱 AI（推荐）**:
```
API Base URL: https://open.bigmodel.cn/api/anthropic
API Key: f8b3db6f0e6c8ed9c7890fe4d3c43be6.6tNmJ4ncX8h3JKdw
Model: glm-5
```

**使用 Anthropic（官方）**:
```
API Base URL: [留空]
API Key: [你的 Anthropic API Key]
Model: claude-3-sonnet-20240229
```

#### 步骤 3: 测试连接
- 点击 **"Test Connection"** 按钮
- 看到 ✅ "Connection successful" 表示配置成功

#### 步骤 4: 保存配置
- 点击 **"Save"** 按钮
- 重启应用

---

### 2️⃣ 使用 AI 功能

#### 打开 AI 面板

**在技能编辑器中**:
1. 点击工具栏上的 **"AI Assist"** 按钮
2. AI 面板会从右侧滑出

## 🎯 AI 功能模式

### 模式 1: 新建技能 (New)

**用途**: 从头创建一个全新的技能

**使用场景**:
- 想要创建新技能但不知道如何开始
- 需要生成技能模板
- 快速生成技能框架

**操作步骤**:
1. 点击 **"AI Assist"** 按钮打开 AI 面板
2. 选择模式: **"New Skill"**
3. 输入提示，例如:
   ```
   创建一个代码审查技能，帮助审查 JavaScript 代码
   ```
4. 点击 **"Generate"**
5. AI 会生成完整的技能内容
6. 预览后点击 **"Apply"** 应用到编辑器

**示例提示**:
```
✅ "创建一个 Git 提交规范检查技能"
✅ "生成一个 API 文档生成技能"
✅ "创建一个 React 组件测试技能"
✅ "生成一个代码格式化技能"
```

---

### 模式 2: 修改技能 (Modify)

**用途**: 改进或扩展现有技能内容

**使用场景**:
- 改进技能质量
- 添加新功能
- 优化描述和说明

**操作步骤**:
1. 在编辑器中打开现有技能
2. 点击 **"AI Assist"** 按钮打开 AI 面板
3. 选择模式: **"Modify"**
4. 输入修改指令，例如:
   ```
   添加更多代码示例
   ```
   或
   ```
   改进技能描述，使其更清晰
   ```
5. 点击 **"Generate"**
6. AI 会基于当前内容生成改进版本
7. 点击 **"Apply"** 替换原有内容

**示例提示**:
```
✅ "添加错误处理示例"
✅ "增加性能优化建议"
✅ "补充最佳实践部分"
✅ "改进文档结构和可读性"
```

---

### 模式 3: 插入内容 (Insert)

**用途**: 在光标位置插入新内容

**使用场景**:
- 在特定位置添加新章节
- 插入代码示例
- 添加补充说明

**操作步骤**:
1. 在编辑器中，将光标放在想要插入的位置
2. 点击 **"AI Assist"** 按钮打开 AI 面板
3. 选择模式: **"Insert"**
4. 输入要插入的内容描述:
   ```
   插入一个使用示例
   ```
5. 点击 **"Generate"**
6. AI 会生成适合当前位置的内容
7. 点击 **"Apply"** 插入内容

**示例提示**:
```
✅ "在此处插入快速开始指南"
✅ "添加一个复杂示例"
✅ "插入常见问题解答"
✅ "添加配置说明"
```

---

### 模式 4: 替换选中 (Replace)

**用途**: 替换选中的文本内容

**使用场景**:
- 重写某个段落
- 改进选中的代码示例
- 优化特定部分

**操作步骤**:
1. 在编辑器中，选中想要替换的文本
2. 点击 **"AI Assist"** 按钮打开 AI 面板
3. 选择模式: **"Replace"**
4. 输入替换指令:
   ```
   重写这段内容，使其更简洁
   ```
5. 点击 **"Generate"**
6. AI 会生成替换内容
7. 点击 **"Apply"** 替换选中文本

**示例提示**:
```
✅ "用更简单的语言重写"
✅ "添加更多细节和示例"
✅ "改为 Markdown 格式"
✅ "翻译成英文"
```

## 💡 使用技巧

### 技巧 1: 提供清晰的提示

❌ **不清晰的提示**:
```
帮我改一下
```

✅ **清晰的提示**:
```
添加 3 个实际使用示例，包括输入和输出
```

---

### 技巧 2: 分步骤生成

不要一次性要求太多内容，分步骤生成：

**步骤 1**:
```
生成技能的基本框架和 YAML 头部
```

**步骤 2**:
```
添加使用说明部分
```

**步骤 3**:
```
补充代码示例
```

---

### 技巧 3: 使用上下文

AI 可以看到当前编辑器内容，利用这一点：

```
基于上面的配置说明，添加一个实际配置示例
```

---

### 技巧 4: 迭代改进

第一次生成不满意？继续改进：

```
这个示例太简单了，给我一个更复杂的实际案例
```

或

```
解释得更详细一些，适合初学者
```

## ⚠️ 注意事项

### 1. API 速率限制

**智谱 AI**:
- 无明确限制
- 建议合理使用

**Anthropic**:
- 按 API Key 类型不同
- 查看你的账户配额

### 2. 内容质量

AI 生成的内容可能需要：
- ✅ 人工审核
- ✅ 适当调整
- ✅ 添加个人经验

### 3. 网络连接

- 需要稳定的网络连接
- 生成过程可能需要 10-30 秒
- 请耐心等待

## 🎯 最佳实践

### ✅ DO（推荐做法）

1. **提供详细描述**: 越详细，生成效果越好
2. **使用示例**: 让 AI 知道你期望的格式
3. **迭代改进**: 不满意就继续调整
4. **结合手动编辑**: AI 生成 + 人工优化
5. **保存常用提示**: 记录有效的提示词

### ❌ DON'T（避免做法）

1. **模糊的指令**: "帮我写个技能"
2. **一次性要求太多**: 分步骤更好
3. **忽略预览**: 总是预览后再应用
4. **不保存工作**: 定期保存你的技能

## 🔧 故障排除

### 问题 1: AI 面板打不开

**解决方案**:
1. 检查 AI 是否已配置（Settings → AI）
2. 确认 API Key 是否正确
3. 重启应用

### 问题 2: 生成失败

**可能原因**:
- API Key 无效或过期
- 网络连接问题
- API 速率限制用尽

**解决方案**:
1. 测试 AI 连接（Settings → AI → Test Connection）
2. 检查网络连接
3. 等待一段时间后重试

### 问题 3: 生成内容不理想

**解决方案**:
1. 提供更详细的提示
2. 使用示例说明你想要的格式
3. 尝试不同的表达方式
4. 分步骤生成
