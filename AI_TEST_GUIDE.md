# AI助手功能测试指南

## 测试前准备

### 1. 启动应用
```bash
cd /d/skillsMN
npm start
```

### 2. 配置API密钥
在使用AI助手之前，需要配置Claude API密钥：
1. 打开应用
2. 进入 **Settings** (设置)
3. 找到 **AI Configuration** 部分
4. 输入你的 **Anthropic API Key**
5. 保存设置

**获取API密钥**：
- 访问：https://console.anthropic.com/
- 创建API密钥（需要Claude Pro或Team订阅）

## 功能测试清单

### ✅ 测试1：打开AI助手面板
1. 在技能编辑器中，点击右上角的 **AI助手** 按钮（魔法棒图标）
2. 验证：侧边栏从右侧滑出，显示 "AI Skill Assistant" 标题

### ✅ 测试2：模式切换
AI助手支持4种模式，验证每种模式是否正常工作：

#### 模式A：新建技能 (new)
1. 选择 **"Create New Skill"** 模式
2. 输入提示词：`创建一个Python代码审查技能`
3. 点击 **Generate** 按钮
4. 验证：
   - 状态显示 "STREAMING"
   - 内容逐步显示在预览区域（流式输出）
   - 完成后显示 "COMPLETE" 状态
   - **Apply** 按钮变为可用

#### 模式B：修改技能 (modify)
1. 先在编辑器中打开一个现有技能
2. 选择 **"Modify Existing Skill"** 模式
3. 输入提示词：`添加一个新的触发条件：当用户提到"测试"时`
4. 点击 **Generate**
5. 验证：AI基于当前技能内容进行修改

#### 模式C：插入内容 (insert)
1. 在编辑器中，将光标放在要插入内容的位置
2. 选择 **"Insert at Cursor"** 模式
3. 输入提示词：`在这里添加一个代码示例`
4. 点击 **Generate**
5. 验证：生成的内容适合插入到光标位置

#### 模式D：替换选中内容 (replace)
1. 在编辑器中选中一段文本
2. 选择 **"Replace Selection"** 模式
3. 输入提示词：`将这段内容改写得更简洁`
4. 点击 **Generate**
5. 验证：生成的内容是对选中内容的重写

### ✅ 测试3：流式输出
1. 启动一次生成任务
2. 观察：
   - 内容是否逐步显示（而不是一次性出现）
   - 状态指示器是否显示 "STREAMING"
   - 光标是否有加载动画

### ✅ 测试4：超时警告
1. 启动一次生成任务
2. 等待30秒以上
3. 验证：如果生成时间过长，应显示黄色警告框：
   - "Generation is taking longer than expected"
   - 提供 **Continue** 和 **Cancel** 按钮

### ✅ 测试5：停止生成
1. 启动一次生成任务
2. 在生成过程中点击 **Stop** 按钮
3. 验证：
   - 生成立即停止
   - 状态变为已取消或错误
   - 可以重新开始生成

### ✅ 测试6：应用生成的内容
1. 完成一次生成
2. 点击 **Apply** 按钮
3. 验证：
   - 内容被插入到编辑器中
   - AI助手面板自动关闭
   - 编辑器内容已更新

### ✅ 测试7：错误处理
1. 故意输入无效的API密钥或断开网络
2. 尝试生成内容
3. 验证：
   - 显示清晰的错误消息
   - 提供 **Retry** 按钮
   - 错误消息说明问题原因

### ✅ 测试8：重试功能
1. 在生成失败后
2. 点击 **Retry** 按钮
3. 验证：重新尝试生成

### ✅ 测试9：键盘快捷键（如果有）
检查是否存在以下快捷键：
- `Ctrl/Cmd + Shift + A`：打开AI助手面板
- `Escape`：关闭AI助手面板

### ✅ 测试10：边界情况

#### 测试10.1：空提示词
1. 不输入任何内容
2. 点击 **Generate**
3. 验证：按钮被禁用或显示提示

#### 测试10.2：超长提示词
1. 输入超过2000字符的提示词
2. 验证：输入框有字符限制或显示警告

#### 测试10.3：并发请求
1. 启动一次生成
2. 尝试立即启动第二次生成
3. 验证：第二次请求被阻止或排队

## 性能测试

### 响应时间
- 打开AI助手面板：< 300ms
- 开始生成（首字节到达）：< 2s
- 流式更新频率：每100-300ms

### 资源使用
- 内存占用：生成过程中应 < 500MB
- CPU使用：空闲时 < 5%，生成时 < 30%

## 已知问题检查

根据代码分析，以下功能需要特别注意：

1. **Timeout检测**：确保30秒超时警告正常工作
2. **流式传输**：验证内容逐步显示，不是一次性出现
3. **模式切换**：切换模式时，之前的生成应该被清除
4. **面板关闭**：关闭面板时，正在进行的生成应该被取消
