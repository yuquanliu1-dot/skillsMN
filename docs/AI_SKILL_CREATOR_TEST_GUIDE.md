# AI Skill Creator - 测试指南

## 🧪 测试步骤

### 1. 启动应用
```bash
npm start
```

### 2. 打开AI Skill Creator
1. 点击 "Create New Skill" 按钮
2. 选择目录 (Project 或 Global)
3. 点击紫色 "AI Create" 按钮

### 3. 测试界面布局

#### ✅ 检查项:
- [ ] 顶部显示渐变紫色背景
- [ ] 显示实际保存路径 (非"Project Directory")
- [ ] 输入框在预览区域上方
- [ ] 按钮位置正确: [Generate] 或 [Stop]/[Retry]/[Create]
- [ ] 预览区域占据大部分空间

### 4. 测试工具调用反馈

#### 测试用例1: 创建简单skill
```
输入: "创建一个简单的代码格式化工具"
```

**预期**:
1. 点击Generate后
2. 立即看到蓝色工具卡片
3. 卡片显示: `🔧 skill-creator`
4. 工具输入参数显示在卡片中
5. 预览区域开始流式显示内容

#### ✅ 检查项:
- [ ] 工具卡片正确显示
- [ ] 工具图标动画 (animate-pulse)
- [ ] 工具名称清晰可读
- [ ] 输入参数格式化显示
- [ ] 蓝色边框和背景

### 5. 测试流式生成

#### ✅ 检查项:
- [ ] 文本逐字/逐块流式显示
- [ ] 紫色闪烁光标跟随文本
- [ ] "Streaming..." 标签显示
- [ ] 滚动条自动跟随内容

### 6. 测试完成状态

#### ✅ 检查项:
- [ ] "Streaming..." 变为 "Complete"
- [ ] 绿色勾选图标显示
- [ ] 光标消失
- [ ] [Create Skill] 按钮可用

### 7. 测试创建流程

#### 步骤:
1. 点击 [Create Skill]
2. 观察对话框关闭
3. 检查skill列表刷新
4. 验证新skill出现在列表中

#### ✅ 检查项:
- [ ] 对话框正确关闭
- [ ] 无alert弹窗
- [ ] Toast通知显示 "Skill created successfully with AI!"
- [ ] Skill列表自动刷新
- [ ] 新skill可点击编辑

### 8. 测试错误处理

#### 测试用例: 网络断开
1. 断开网络
2. 点击Generate

**预期**:
- [ ] 显示错误信息
- [ ] Retry按钮可用
- [ ] 可重新尝试

### 9. 测试取消操作

#### 步骤:
1. 开始生成
2. 点击 [Stop] 按钮

**预期**:
- [ ] 生成停止
- [ ] 保留已生成内容
- [ ] Retry按钮显示

### 10. 测试界面响应

#### ✅ 检查项:
- [ ] 对话框高度 85vh
- [ ] 内容可滚动
- [ ] 窗口大小调整时布局正确
- [ ] 移动端响应式 (如果支持)

## 🎨 视觉验证

### 颜色检查
- [ ] 头部渐变: 紫色到靛蓝色
- [ ] 工具卡片: 蓝色系 (blue-50/200/600/900)
- [ ] 生成按钮: 紫色渐变
- [ ] 停止按钮: 红色 (red-500)
- [ ] 重试按钮: 琥珀色 (amber-500)
- [ ] 创建按钮: 绿色 (green-500)

### 动画检查
- [ ] 工具图标脉冲动画
- [ ] 流式光标闪烁
- [ ] 按钮hover效果
- [ ] 对话框打开动画

**检查**:

## 🐛 已知问题检查

### 问题1: 工具调用不显示
**排查**:
1. 检查浏览器控制台
2. 确认后端发送了tool_use事件
3. 验证前端正确处理

### 问题2: 流式中断
**排查**:
1. 检查网络连接
2. 查看后端日志
3. 确认AI API正常

### 问题3: Skill创建失败
**排查**:
1. 验证YAML frontmatter正确
2. 检查skill name解析
3. 确认目录权限
